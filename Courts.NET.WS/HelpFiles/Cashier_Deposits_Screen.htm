Cashier Deposits Screen

From this screen a cashier is able to process their deposits to either the safe or bank once they have performed Cashier Totals.

 

Cashier Deposits user rights

 

The following are user rights which apply to the Cashier Deposits screen:

 

  • Cashier Deposit - Will allow the user access to the Deposit Screen. This should not be given to cashiers. Only supervisors should be given access to this screen.

  • Employee appears in Cashier Deposits - Anybody who acts as a cashier whether they are set up as a cashier or not must have this user right. If it is not ticked their name will not appear in the deposits screen and the deposit will not be possible.

Sessions

 

A session is defined as a period of time from when a cashier makes their first transaction which causes their ‘Amt Available for Deposit’ to change to the point at which they record their Cashier Totals.

Head Office Vs Branch

 

  • If the screen is opened by a user at the head office branch a drop down will appear to allow the user to select the branch they would like to select.

  • This will default to the current branch and load the list of cashiers accordingly. If a new branch is selected the cashiers for that branch will be loaded.

  • If not the head office branch the drop down will be disabled and only the cashiers for the current branch can be loaded.

Definitions of fields

 

The data grid in the upper part of the screen will be populated with a list of all cashiers in the selected branch. It will display a summary of their current status.

 

Below is a specific description of the fields displayed:

 

  • Amt Available for Deposit – this will be a combination of income received since the cashier last totalled and income that has been totalled but not yet been deposited. This will also include the cashier’s allocated float. Money that the cashier has deposited to the safe will fall into the category of already deposited and will therefore not be included in the 'Amt Available for Deposit'.

  • Transferred to Safe – this is the amount that the cashier has transferred to the safe in this session. Money transferred to the safe after the cashier totals but before they start a new session should not be included in the 'Transferred to Safe' column.

  • Allocated Float – This is the amount that was allocated to the cashier as a float at the beginning of the session. A float allocation is effectively the reverse of a transfer to the safe, as we are transferring money from the safe to the cashier.

  • Totalled –  This will either be set to ‘N’ or ‘Y’. This should be set to ‘Y’ once a cashier is considered totalled and they have recorded their totals via the Cashier Totals screen, and have not yet started a new session. As soon as a new session has begun the cashier is no longer considered totalled and this should be set to ‘N’.

 

The data grid in the lower part of the screen (deposits) is a breakdown of deposits per cashier.

 

  • Pay Method – This is a list of the Pay Methods.

  • Amt Available for Deposit - This is a breakdown of the amount available for deposit by pay method.

  • Amount Banked - If the cashier ‘totalled’ column is = N, then the supervisor will not be able to deposit any money to the bank. This is the column where the supervisor puts in the totals for the amount to deposit to the bank. If the totalled column = Y then this field becomes editable. Whatever amount is entered in this column will be deducted from the 'Amt Available for Deposit'. The cashier will not be able to log in until the 'Amt Available for Deposit = ‘0’ for each pay method.

  • Amount to Safe - This field may be used by the cashier to deposit money to the safe during the day. This field may also be used to deposit money to the safe at the end of day (once totalled) instead of the bank. This is also the field that would be used to allocate a float to the cashier at the beginning of their session . This is achieved by entering a negative amount in this field. The Allocated float field should be updated with the float amount.

Making a Cashier Deposit

 

  • Deposits can be made in this screen from a cashier to the safe or to a bank.

  • The specific bank deposited to will be a branch level parameter which will need to be maintainable in the Branch Maintenance Screen (under the Bank Deposits tab).

  • The dummy “SAF” cashier cannot make a deposit to the safe obviously but only to a bank.

  • To make a deposit for a cashier the cashier’s record in the upper data grid should clicked on. This will populate the lower data grid with a breakdown of the cashier’s amount available for deposit by payment method.

  • The 'Amt Available for Deposit' shown in the upper data grid will be broken down into all of the available payment methods and displayed in the lower data grid in a read only column to guide the user showing the maximum amount that can be deposited for each payment method.

  • Money cannot be deposited to the bank unless it has been totalled. Therefore, if the cashier has totalled then the user should be able to enter the amount of the available deposit total for each payment method that is being banked in the 'Amount Banked' field.

  • If the cashier has not totalled it will not be possible to make a deposit to the bank and the 'Amount Banked' field will be read only (and set to zero) accordingly.

  • The user should be able to enter any value up to and including the 'Amt Available for Deposit' for the current payment method in to the 'Amount to Safe' field which in this case is not read only.

  • If the amount being deposited to the bank is less than the 'Amt Available for Deposit', the difference should automatically be entered into the 'Amount to Safe' field that will be read only.

  • If the amount being deposited is greater than the 'Amt Available for Deposit'  then the entered figure will be ignored and defaulted to the amount available.

  • For deposits to the bank a lodgement slip reference number is mandatory. The same lodgement slip number can be used for each payment method on the same deposit, but not for deposits recorded on separate occasions.

  • When the save button is clicked the cashier deposits will be recorded and the lower data grid will be cleared.

  • The upper data grid will then be refreshed to make sure that the figures reflect the deposits just made. Those changes will depend on what kind of deposit has been made. Any deposit will reduce the cashier’s 'Amt Available for Deposit' by the amount of the deposit made.

  • If a deposit to the safe was made then this will be reflected in the 'Transferred to Safe' column if the cashier has not totalled.

  • If the cashier has totalled the 'Amt Available for Deposit' by the dummy “SAF” cashier will be increased by the amount of the deposit.

  • Therefore the 'Amt Available for Deposit' for the dummy cashier “SAF” is not the sum of the value in the other cashier’s 'Transferred to Safe' columns; rather it is the amount actually in the safe minus the amount that has not been totalled (namely - the sum of the value in the other cashier’s 'Transferred to Safe' columns).

Making a Deposit from the Safe

 

  •  Since the dummy cashier “SAF” also appears in the upper data grid, it can be selected and deposits can be made from it. The totalled flag is not meaningful for this cashier and will be ignored.

  • For this cashier it will only ever be the 'Amount Banked 'column in the lower data grid that can be written into. The only difference will be that regardless of the amount entered into the 'Amount Banked' field, the 'Amount to Safe field will remain set to zero and read only.

  • When a deposit is saved from the safe to the bank the lower data grid should then be cleared and the upper data grid refreshed as before to reflect the change.

  • The 'Amt Available for Deposit' for the SAF cashier should have reduced.

  • The 'Transferred to Safe' columns for all other cashiers should remain unchanged because that is un-totalled money and therefore was not part of the dummy cashier “SAF” 'Amt Available for Deposit'.

Reversing Safe Transfers

 

  • It is a country specific requirement that any money in the cashiers’ Transferred to Safe' column must be reversed (i.e. transferred back to the cashier) before the cashier can total. This is controlled by the Country parameter ‘Allow Outstanding Safe Deposits’.

  • Thus a cashier can never have a value in the 'Transferred to Safe' column and a totalled flag set to “Y”.

  • To reverse any amount in the 'Transferred to Safe' column the user must right-click this field and select ‘Reverse safe transfer’.

  • This will move the figure from the 'Transferred to Safe' column and add it to the 'Amt Available for Deposit' column for the cashier.

  • The cashier should then be able to perform Cashier Totals. Then the money can  either be deposited back to the safe or to the bank (or a combination of the two as described).

Debit and Credit Card Transactions

 

Debit and credit card transactions can each be electronic or manual and consolidated or not consolidated. Electronic or manual just refers to the type of machine used to process the transactions.

 

If they are consolidated, all payments for that branch will be consolidated and posted to FACT2000 as a single amount.

 

These options are country specific and should be controlled by country parameters. Depending on how these options are set, the behaviour of the deposits screen for these payment methods will be slightly different.

 

The following combinations of options will lead to the behaviour described in the cashier deposits grid for these payment methods:

 

On the totalling of the Cashier the following should occur:

 

  • Electronic / non-consolidated - the amount being banked field will be automatically populated with the total amount available for deposit for that payment method.

  • Non- Electronic/ non-consolidated - the user is able to input into the amount being banked field and any remainder will be used to populate the amount being transferred to the safe.

  • Non-Electronic/ Consolidated - Amount being banked field is read only and the user is able to manually input the amount being deposited to the safe (which cannot exceed the amount they have available for deposit).

  • Electronic/ Consolidated - User can only transfer to Safe. Where the screen refers to the details for the Safe, the system should auto populate the ‘Amount being Banked’ field.

 

Where the Cashier has not been totalled the following should occur:

 

  • Electronic / non-consolidated and electronic / consolidated - for these two options the user should not be able to access any of the fields for Credit/Debit Cards.

  • For the other two options (Non-Electronic/ Non-Consolidated and Non-Electronic/ Consolidated) the user can transfer to Safe but has to transfer back before the cashier can be totalled.

Negative Deposits

 

When the Pay Method reference (under Code Maintenance, category ‘FPM’) is not set to ‘1’,  negative amounts cannot be entered for the Bank amount but they can still be entered for the Safe amount.

 

When the Pay Method reference is set to ‘1’, negative amounts can be entered for the Bank amount, and Safe amount. There are different criteria that must be taken into consideration as to when negative deposits can be made. The below describes the criteria:

 

  • During a Cashier session – Cashier should be able to make positive safe deposits and negative safe deposits (for example a float amount).

  • After Cashier totalled – Cashier should be able to make positive safe and bank deposits, and also negative safe and bank deposits.

  • After all amounts deposited (start of a new session) – Cashier should be able to make negative safe and bank deposits.

Foreign Pay Method  Deposits

 

Please note that for any foreign currency pay method, the amount taken as income should be converted into its local equivalent when the total ‘Amt Available for Deposit’ is calculated for the cashier.

Refunds/Corrections, Disbursements affect on Amt Available for Deposit

 

In the Cashier Deposits screen, please note that any Pay Methods that were used to process a correction/refund or disbursement, their value is reduced by the refunded/corrected or disbursed amount. This will also have an affect on the ‘Amt Available for Deposit’ field for the cashier which will be reduced by any refunds/corrections and disbursements.
