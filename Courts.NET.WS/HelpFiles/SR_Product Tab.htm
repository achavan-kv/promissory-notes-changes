The Product Tab

The Product tab at the bottom of the screen will be populated with any product from the Product List. A Product is selected in the list a red RED label next to the purchase date in the Product tab will indicate if product selected is not covered by warranty. The label will say "NO WARRANTY". If the item is under Warranty then no message will show. If in extended warranty then the extended warranty box will be ticked.

 

The Product tab will only be enabled when a product is selected.

 

The following fields will be pre populated and never editable:
  • Purchase Date
  • Product Code
  • Stock Location
  • Description of Product
  • Logged by
  • Instant Replacement - This will be ticked if the item is under Instant Replacement.
Fields not pre populated and editable (within certain conditions) are:
  • Model No
  • Serial Number
  • Date logged - This default to the day the request was raised but can be changed to an earlier date)
  • Product Received date
  • Service Location - Only visible when enabled by the country parameter 'Service Location'. The descriptions for Service Locations should be defined in codes maintenance (SRSERVLCN).
  • Repair Estimate - Required if the 'Allocation tab' is enabled. It will only be enabled if an estimate was required on the 'BER Popup' screen
  • Product Collected date - This will be disabled until a Resolution date is set in the Resolution tab.
  • Extended Warranty - If the product is out of 1st year manufacturer warranty but still in the extended warranty period the system will tick this field. This field can be changed with authorisation and it can only be changed from 'Y' to 'N'. Not from No to Yes.
  • Damaged by deliver - When this is ticked, a deposit will not be required from the customer. Additionally the soft script popup will not appear when pressing the repair button.
  • Goods on Loan - To be ticked if good have been loaned to the customer as a temporary replacement during repair.
  • Deposit Required - Required if allocation is enabled. A deposit is only required when an item is out of Warranty and the country parameter 'Service Deposit' is enabled. The user will enter the amount of the deposit required here. Payment will be made via the payment screen. This is discussed in the Payment Screen topic.
  • Repair Button - This is enabled when the status of the Service Request is either new or allocated. If the damaged by deliver box has NOT been ticked, the Repair button will launch the 'Soft Script popup' screen. If the damaged by deliver is true then the cause for the repair is already known.
