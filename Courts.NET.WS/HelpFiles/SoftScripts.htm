Soft Script Popup

This will be a popup dialogue opened by the ‘Repair’ button on the Service Request screen. A series of questions loaded from the code maintenance category ‘SRSCRIPT’ will be loaded. Each question will have two tick boxes to indicate a ‘Yes’ or ‘No’ answer. The user will read through the script with the customer in order to decide whether to continue with the SR or close the SR.

The comments will be added to the same comments for the SR visible as the ‘Comments’ tab on the Service Request screen.

The date, time and employee number will be automatically added to each comment.

The ‘Service Evaluation’ list of options will only be enabled once all the questions are ticked with either a ‘Yes’ or ‘No’ answer.

The ‘Service Evaluation’ will be mandatory and must be entered to enable the ‘Allocate’ and ‘Close SR’ buttons.

 

This will be set to one of:

  • No Fault Found (NFF) - only the ‘Close SR’ button will be enabled (status ‘C’ will be returned)
  • Misuse by the customer (MIS) - will enable both buttons (status ‘D’ will be returned)
  • Event or terms not covered (EVC) - will enable both buttons (status ‘D’ will be returned)
  • Warranty Covered (WAR) - will enable both buttons (status ‘A’ will be returned)

The ‘Service Evaluation’ will be passed back to the Service Request screen where it will be displayed as a read only field on the ‘Product’ tab.

The ‘Allocate’ button will save any comments and will set the column Service Request status to either ‘A’ or ‘D’ according to the Service Evaluation selected.

The ‘Close SR’ button will save any comments and close the SR by adding a resolution date and setting the resolution description to ‘Save a Call’ (SAC). In this case the column Service Request Status will be set to ‘C’.
