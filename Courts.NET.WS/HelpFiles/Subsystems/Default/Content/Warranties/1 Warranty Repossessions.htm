1 Warranty Repossessions.

 

When the product linked to a warranty is repossessed the system need to make sure the following takes place:

 

  • The customer is refunded the unexpired portion of the warranty via a CRE or CRF adjustment
  • The warranty is not redelivered if the repossession is redelivered.
  • AIG is notified of the repossession via warranty reports in CoSACS

1.1 The Repossession process

 

  1. Customer returns product “x” months since delivery
  2. When returning the product the user will get a message informing them that the Product is linked to a warranty.
  3. The warranty will be automatically returned with the product.
  4. The warranty return item number  will be based on the return code tables in COSACS. The return codes are necessary to work out the rebate that can be claimed back from AIG for the unexpired portion of the warranty. The customer will also receive a credit to their account for the unexpired portion of the warranty. This will be in the form of transactions CRE or CRF. 
  5. The transaction codes are set up by default in the CoSACS upgrade in transaction type maintenance with the correct interface account numbers.
  6. Process the repossession as normal.
  7. In Account details you will see the details of the repossession and the transaction.
  8. A warranty is always repossessed with a zero value. This is because the customer is still expected to pay for the warranty. However we cannot expect the customer to pay the unexpired of the warranty. The customer is credited for the unexpired portion of the warranty based on the Warranty Return Codes set up in CoSACS. Any changes to return codes and % associated with them needs to be requested through CoSACS Support.

 

Redelivery after repossession

Please note that warranties are not redelivered if a repossession is redelivered. This is because the redelivered item is considered as second hand by the Insurers and the item cover is terminated.
