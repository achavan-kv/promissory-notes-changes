6 Warranty Renewals

 

The following section is a guide to Warranty Renewals. Before getting started there a few concepts that should be made clear.

 

What is a Warranty renewal?
  1. A Renewal is a 12 month Warranty that will be offered to Cash or Credit Customers after their existing Supashield Warranty expires.
  2. Renewal Warranty codes all end in a ‘1’ indicating this one year period.
  3. Renewals will only be offered on 3 year Supashield Contracts. 

 

How will the customer be made aware if they have warranties that fit the criteria for a renewal?

 

There will be 2 methods of informing the customer of the renewal offer

 

  1. A letter will be sent when the customers account fits the criteria for a renewal offer
  2. A Renewal prompt will pop up on the payment screen if the customer has any accounts which fit the relevant criteria

 

What are the Criteria for a Renewal?

 

  1. The original warranty must have been a 3 year Supashield
  2. The customer settles an account within a certain time period before the existing warranty is due to expire. This is controlled by a country parameter called, ‘ Warranty Expiry settlement prompt days’
  3. The customer makes a payment on an account but may have other settled accounts with warranties that have expired a specified number of days ago. This is controlled by the parameter, “ Maximum days after original warranty expired”
  4. The Customer makes a part payment on an account and the warranty is due to expire within a certain time period. This is controlled by the parameter ‘Warranty Expiry days’.

 

But our cashiers don’t do sales?

 

  1. There is a user right called, “Allow cashier to do renewal sale”. If this is set to false then the cashier will only receive an information message about the opportunity the customer has to renew their warranty. The customer will be asked to go to a Salesperson who will then be able to process the renewal for them.

 

Won’t the customers get irritated if they don’t want the renewal but continually get reminded about it?

 

  1. For this very reason the customer will only be reminded 3 times about a warranty that is up for renewal. After the third time there will no longer be a pop up at the payment desk. However there will be an option on the payment menu called, “Check expiring warranties”. This will bring back a list of all the warranties currently up for renewal.

6.1 Warranty Renewals Process Flow

6.1.1 Letters

The parameter “Warranty Expiry letter generation in months” can either be set to a number of months or ‘N’. If set to ‘N’ then no letters will be sent.

Otherwise if the parameter is set to a number of months then CoSACS will generate a letter (code = WR) for that customer when the original Supashield is going to expire within that time period.

Only one letter is ever sent to the customer.

6.1.2 Customer makes a payment at cash desk

The customer can either come in to settle an account or make a part payment on an account.

Each of the below checks must be true for each listed scenario:

 

Scenario 1: Settling an account which has a Warranty on it. The customer has no other accounts due for renewal

On tabbing out of the “Amount tendered” field:

 

  • Has the account being settled got a 3 year Supashield on it?
  • Is that 3 year Supashield due to expire within <Warranty expiry settlement days> OR Is the Warranty due to expire within <Warranty expiry days>
  • Has the Customer been prompted LESS 3 times to purchase this renewal already?

 

Scenario 2: Settling or taking part payment an account which has does not have a warranty on it or the warranty is not up for renewal. The customer has another account with a warranty

On tabbing out of the “Amount tendered” field:

 

  • Has the settled account got a 3 year Supashield on it?
  • Is that 3 year Supashield due to expire within <Warranty expiry settlement days> OR Is the Warranty due to expire within <Warranty expiry days> OR has the warranty already expired and falls with the period <Maximum prompt days after original warranty expired>
  • Has the Customer been prompted LESS 3 times to purchase this renewal already?

 

Scenario 3: Taking part payment on an account which has a Warranty on it. The customer has no other accounts due for renewal

On tabbing out of the “Amount tendered” field:

 

  • Has the account got a 3 year Supashield on it?
  • Is that 3 year Supashield due to expire within the period <warranty expiry days>
  • Has the Customer been prompted LESS 3 times to purchase this renewal already?

 

If either of the above scenarios is true the cashier will be prompted with a message that a renewal is available.

If the user right, ‘Allow cashier to do renewal sale’ is set to False then the following message will pop up:

“Please ask the customer to go to the sales person for continued cover on item”

This is where the process ends for the cashier.

If the user right, ‘Allow cashier to do renewal sale’ is set to TRUE then prompt will appear allowing the user to continue with the sale.

The cashier will enter the warranty contract number (unless the system auto generates this). On pressing “BUY” the cashier will be redirected to the New sales Order Screen where they will have to Choose the account type (C or H). The warranty will be added to the new order automatically. This new account bypasses credit sanctioning.

The Warranty contract will print out the future delivery date of the renewal warranty. This will only be the day after the original warranty expired.

6.2 Delivery of a renewal warranty

 

A warranty renewal will be delivered a day after the original warranty is due to expire. If the renewal has been offered on a warranty that had already expired then the renewal will be delivered on the same day it is sold.

6.3 Reports

Warranty Renewal Reports are available from the Warranty Reporting Screen in .NET. There are two reports:

  • Accounts due for renewals
  • Renewal Sales

6.4 Warranty Set Up in FACT2000

 

The warranty renewal product codes are set-up in the same manner as standard warranty product codes with the exception that the warranty reference (field number 21) must be set to ‘XX’. This is the code that has been allocated to this type of warranty.

 

Therefore, select option 0261 (Product Details) and set-up the required Warranty Renewal product codes(click here for screen shot)

 

As this type of warranty is only valid for one year and to conform with the Price Banding rules, each product code allocated must end in a ‘1’.

 

In addition to setting up all the Warranty Renewal product codes, it is also necessary to create the Warranty Renewal price banding in the same manner as that for Instant Replacement processing. (Click here for screen shot)

To achieve this, you first need to ensure the Price Banding is ‘switched on’ by selecting option 0271 (General Details) and setting the ‘Price Banding’ indicator (field 13) to ‘Y’ as shown below.

Once this has been completed, the actual Price Banding details can be created using option 02276 (Price Banding Details) (click here for screen shot).


As code ‘XX’ is a pre-defined code, this code does not need to be set-up using option 02775 (Warranty Details).

 

The warranty term must be set to ‘1’ .

6.5 Marking goods as Renewable in Fact2000

 

As only certain products are deemed as ‘warranty renewable’, the product maintenance screen has been amended to include the necessary indicator for identifying the relevant products.

 

This new field (Number 11 – Warr. Renewal) is only available to type ‘3’ product codes (i.e. ‘stocked’ products’). Therefore, select option 0261 (Product Details) and set the indicator to ‘Y’ for all Warranty Renewable products as indicated here.

 

This new ‘Warranty Renewal’ field has been added to the Product Amendment file created using option 02431 which is input to CoSACS on a daily basis.

6.6 Interfaces

 

FACT2000 will send through an indicator against each product if the product is renewable. This will be an ‘Y’. This will be sent through on the product file.

The CoSACS interface to Fact2000 won’t change because FACT2000 will be identifying all Renewals 

This document is to be used as an end user userguide. It is not a technical specification. The document will describe the procedure the user should follow in order to sell a Warranty on Credit when the items are all sold on cash or cash & go and the customer does not have enough cash to purchase the warranty. This applies to Supashield and Instant Replacement Warranties.
