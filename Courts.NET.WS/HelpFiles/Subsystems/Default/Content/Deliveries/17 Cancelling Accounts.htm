17 Cancelling Accounts

This screen is where the user can cancel accounts

The system uses the following logic for cancelling accounts

17.1.1 Accounts without payment and delivery

 
  1. Input the account #
  2. Select reason code
  3. Click ‘Cancel Account’ Button
  4. The system will do the following in the background:
  5. Adjust stock  amount in CoSACS (Increase the  Available by the quantity ordered amount)
  6. Remove all lines  (stock and non stock)
  7. Zero out the following fields in Account Details screen, Last Instalment, Total Instalment, Service Charge, To Follow.
  8. Change status code to S
  9. Application status = cancelled
  10. Update ‘Last changed by’ with the users information 

17.1.2 Accounts with payment and no delivery

 
  1. Input the account #
  2. Select reason code
  3. Click ‘Cancel Account’ Button
  4. System process:
    • Transfer all payment postings individually to the Branch Sundry Credit screen 
    • remove all lines 
    • Adjust stock  amount in CoSACS (Increase the  Available by the quantity ordered amount ) 
    • Zero out the following fields in Account Details screen, Last Instalment, Total Instalment, Service Charge, To Follow.
    • Change status code to S
    • Application status = cancelled
    • Update ‘Last changed by’ with the users information

17.1.3 Accounts without payment but have delivery (after collection for all stock items have been notified) 

 
  1. Input the account #
  2. Select reason code
  3. Click ‘Cancel Account’ Button
  4. System process:
  5. remove all line items 
  6. adjust stock  amount in CoSACS (Increase the  Available by the quantity ordered amount ) 
  7. zero out the following fields in Account Details screen, Last Instalment, Total Instalment, Service Charge, To Follow
  8. Change status code to S
  9. Application status = Account cancelled
  10. Update ‘Last changed by’ with users information

17.1.4 Accounts with payments and delivery (after collection of all stock items has been notified)

 

The user will have to refund/transfer to sundry any payment and process the collection of the goods. Providing the outstanding balance on the account is zero the user can then cancel the account. The process to be as follows:

  1. Input the account #
  2. Select reason code
  3. Click ‘Cancel Account’ Button
  4. System process:
  5. remove all line items 
  6. adjust stock  amount in CoSACS (Increase the  Available by the quantity ordered amount ) 
  7. zero out the following fields in Account Details screen, Last Instalment, Total Instalment, Service Charge, To Follow
  8. Change status code to S
  9. Application status = Account cancelled
  10. Update ‘Last changed by' with users information

17.2 Reversing a Cancelled Account

An option to be added to the current screen 'Reverse Cancelled Account' with reason code drop down box. The system will change the Status Code to 1, the Application Status to "Reopened Account" and if a credit account add the SD (with the relevant amount) to the line item. The system should also log the user who reversed the cancelled account with the information visible in the employee tab. The user can then add line item etc to the account.
