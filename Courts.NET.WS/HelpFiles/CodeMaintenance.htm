Code Maintenance

This screen is used to determine codes that appear in various parts of the system. Most drop-downs are populated from the information displayed in the screen.

Code – You should know the area of the system you are affecting before you add or remove extra codes. Make sure that codes are distinct for each category

Code Description – You can change the description of existing codes if it helps local users get a better understanding of the system, for instance, language

Sort Order – this should be used for those who want items in drop-downs to appear in a particular order. The lower the sort order the higher up the drop-down the code would appear. For instance for FPM – fintrans payment methods if you wanted payment methods to appear in a particular order in the payment screen you should allocate lower numbers to those items you wish to appear higher in the list.

Reference- this is occasionally used to allow extra functionality of the system. For instance with Cashier deposits to allow negative cashier deposits for that particular payment method (category you should add a 1 to the reference field. For staff categories a reference of 1 will mean that the staff type will appear in the Telephone Action and Allocate Account screens. A reference of 2 will mean that staff type will be eligible for commission paid on arrears
