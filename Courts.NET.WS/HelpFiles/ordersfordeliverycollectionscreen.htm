Orders for Delivery / Collection Screen

After Delivery Authorisation all accounts which are in a status of  ”DA, Awaiting Scheduling” should appear in this screen. This includes items that have been returned in the Goods Return screen.

 

This screen enables items to be released and printed onto a Picklist for scheduling and also print ‘Delivery Notes’ that were previously selected for Immediate Delivery from the New Sales Order screen.

 

 

Orders for Delivery / Collection user rights

 

The following are the user rights which apply to this screen:

 

  • Orders for Delivery – View :should only allow search and view results functionality.

  • Orders for Delivery – Amend: should allow full access to release orders, print picklists etc.

Selection Criteria and their definitions

 

  • Branch/Warehouse – this is the branch where the delivery of the goods will take place from.

  • Delivery Area – you should be able to run a search by Delivery Area. On entering the screen this should default to ‘ALL’.

  • Product Category – there are 3 basic categories (IT, Furniture and Electrical). The second drop down should allow you to further breakdown the chosen category. I.e. Electrical into vision, audio, major whites etc.

  • Account Number – you have the option to enter an account number.

  • Order Dates – select the date range from when the order was taken.

  • Delivery Process - Choose if the list should load for immediate delivery or scheduled items.

  • Include – limits the load to either deliveries or collections. The default is ‘Delivery’ checked only.

Retrieved data from the search

 

  • Delivery Area – the area where the goods are to be delivered.

  • Delivery Note Number – should be allocated after Delivery Authorisation.

  • Undelivered Flag – (from A – Z), and should be set for items which were previously removed from a delivery.

  • Privilege Club – set to ‘Yes’ if the customer is a privilege club member.

  • Date Ordered – should be the date the order was taken.

  • Item Description – a description of the product.

  • Last Updated By – should record the name of the last person who changed the customer account or order.

  • Del or Col – indicator to show if this is a delivery or collection.

  • Released – All items that are in stock should automatically be ticked to release the item. Items that are un-ticked by default are where there is an out of stock situation,  which is stock status (2 or 3) and/or if the expected delivery date is a date in the future. Please note, that the stock can be manually released by ticking the checkbox.

  • Stock Status – there are 4 possible status’s that you should take note of:

  1. IS – In Stock

  2. OR – On Order (set when the amount on order => the amount on the customer order).

  3. NS – Out of Stock

  4. SO – In stock, but on someone else’s schedule.

For items that require Immediate Delivery

 

  1. Search for an account using the search parameters or enter the account number.

  2. Under ‘Delivery Process’, make sure that the ‘Pickup/Immediate’ delivery option is ticked.

  3. Click on the ‘Load’ button. The results should be loaded and you should notice that the ‘Delivery Note’ number is already visible. The delivery note at this point should be printed before the items can be fully delivered.

  4. The ‘Print Delivery Note’ button should become enabled, and only becomes enabled for Immediate Deliveries. The ‘Print Picklist’ button should be disabled.

  5. Make sure that the ‘Released’ column is ticked for the items and click on the ‘Print Delivery Note’ button.

  6. The order should disappear from the screen and the delivery note should be printed for the items.

  7. Please note that a delivery note can only be printed once from this screen. After the note is printed this order should no longer appear in this screen.

  8. You should now be in a position to proceed with the immediate delivery of the order.

For items that require scheduling

 

  1. Search for  an account using the search parameters or enter the account number.

  2. Make sure the Scheduling option is ticked.

  3. Click on the ‘Load’ button. You should notice that the ‘Print Delivery Note’ button is dimmed. This is because you will have to schedule the order onto a delivery load before printing the delivery note.

  4. From this screen the user has the option to print a picklist.

Printing the items requiring scheduling onto a Picklist

 

  1. You should make sure that the ‘Released’ column is ticked for the items you require to be printed onto the Picklist.

  2. Click on the ‘Print Picklist’ button.

  3. This should print all items that you have ‘released’ onto the picklist. (please see the section below on Picklists for more information).

Picklists

 

  • Picklists are generated from the Orders for Delivery / Collection screen.  

  • Each picklist will have a unique reference number.

  • The item in the line will need to be released before the pick list can be printed. Items are not released if they are out of stock or the item already appears on another pick list. This release option can be ticked to release the items.

  • Once a pick list is printed the account will have the following status attached to it: PLP – Pick list printed awaiting scheduling.

  • Items printed onto a picklist but not yet scheduled should still appear in the Orders for Delivery / Collection screen.

  • The order will still appear in the Orders for Delivery / Collection screen but the column ‘Picked’ in the results will be set to ‘Yes’.  

  • Items already on a pick list but NOT scheduled can be added to other pick lists, however if the item is added to a new pick list then the item will be removed from the original pick list.

  • A new delivery note Number should be generated for an item, in the situation where it cannot be released while other items on the same delivery note are released onto a Picklist. The ‘Undelivered Flag’ should also be set to ‘A’.

  • Pick lists can be amended in the Amend / Reprint Pick list screen.

Items removed from a schedule (from Delivery Scheduling screen)

 

  • Items removed from a schedule should re-appear in this screen, ready to be re-scheduled.

  • The ‘Picked’ column should be set to ‘No’ and you should be able to print the items onto a new picklist.

  • Items removed from a schedule in the ‘Delivery Scheduling’ screen should have their Undelivered Flag set to ‘R’ in Orders for Delivery/Collection screen.

 

Return to index

 

Items removed from a Delivery Note (from Delivery Notification) screen

 

  • Items removed from a delivery note, should re-appear in this screen, ready to be re-scheduled.

  • The ‘Picked’ column should be set to ‘No’ and you should be able to print the items onto a new picklist.

  • Items removed from a delivery note in the ‘Delivery Notification’ screen should have their Undelivered Flag set to ‘A’ in the Orders for Delivery/Collection screen.

  • All the above also apply when a Delivery Note is removed from a load.
