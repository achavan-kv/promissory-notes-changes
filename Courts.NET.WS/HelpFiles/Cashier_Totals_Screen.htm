Cashier Totals Screen

In this screen cashier reconciliation is prepared and the cashiers account for all takings at the end of the day.

 

Cashier Totals user rights

 

The following are user rights which are needed for the Cashier Totals screen:

 

  • Cashier Totals – Print Cashier Totals per branch: In Cashier Totals there is tab called ‘Branch’.  With this user right the user can print cashier totals per branch and entering a date range.

  • Cashier Totals - Print Cashier Totals per employee: This will allow the user to print totals by employee.

  • Cashier Totals – Save Unreconciled cashier totals: This user right would be given to a supervisor to save and record reasons for any differences that may occur when a cashier totals.

  • Cashier Totals – Summary Tab: This will allow the user to view the Summary tab in the cashier total screen. This is a summary of totals recorded per branch per payment type.

Performing Cashier Totals with an amount transferred to safe

 

There is a country  parameter which determines whether or not a cashier is permitted to cash up while they have money in the safe.

 

This Country Parameter can be found under ‘Country Maintenance’, ‘Cashier’ and is called: ‘Allow Outstanding Safe Deposits’.

 

For some countries it is a requirement that this money must be returned to the cashier effectively reversing the deposit before the cashier can total. If this is the case and the country parameter has been set to FALSE the user should be presented with the message:

 

“Outstanding Deposits to the safe of (amount) must be reversed in the Cashier Deposits screen before Cashier Totals can be saved for this cashier”.

 

For countries that do not have this requirement, the money in the safe will be subtracted from the amount the cashier is expected to total (via the deposits column as normal).

Performing Cashier Totals where cashier totals match system totals

 

  • Clicking on the ‘Load’ button should populate the Pay Methods where the cashier should enter their Cashier Totals for each pay method.

  • Once entered and the save button is clicked, the system will compare the cashier totals to the system expected totals. Please note that the totals you enter should be the income for each pay method subtracting any corrections, refunds and disbursements made using that pay method. This is the amount you have remaining in your till for that pay method.

  • If there are no differences between the values entered by the cashier and those held within the system, no further action is required and a message stating “Totals match entered amount” should appear in the bottom left of the screen.

  • A print box should appear which on clicking to print should print the ‘Cashiers Daily Reconciliation’.

  • Another prompt should appear: “Outstanding income must be deposited before other functions can be used”, which should force the cashier to go to the Cashier Deposits screen where the amounts can be deposited to either the bank or safe.

Performing Cashier Totals where cashier totals do not match system totals

 

  • Once the cashier has entered their totals and clicked on save and the totals do not match the system expected totals, then the prompt: “Your total does not match for the following pay methods (list of Pay methods)” should appear.

  • The cashier should be able to click on the ‘Cancel’ button if they wish to re-check their entered totals.

  • Clicking on the ‘OK’ button, a ‘Authorization Prompt’ box should appear enabling the supervisor to investigate the problem.

Definitions of fields on the Cashier Totals screen:

 

  • Pay method – List of Payment Methods.

  • System Value - For each pay method the System Value should be the amount taken as income (- SUBTRACTING any refunds or corrections) on that pay method.

  • User Value – The User Value should reflect the total of what the cashier entered in the Cashier Totals screen for each pay method.

  • Deposit Value - The Deposit Value should be the amount Allocated as Float – any Cashier Disbursements that were taken using that pay method.

  • Difference – This should be the difference between the actual system value (which takes into consideration the Deposit Value) and the User Value for the pay method.

  • Reason – it is mandatory that a reason is entered into this field before Cashier Totals can be saved for a cashier.

  • System Total field - should ONLY be the total income that you took for each of the pay methods reduced by the refunds or corrections you performed on that pay method.

  • Deposit Total field - should be the total amount Allocated as Float – any Cashier Disbursements that you took.

  • User Total field - should reflect the total of what you entered in the Cashier Totals screen for each pay method.

  • Total Difference field – This should be the total difference between the actual system total (which takes into consideration the Deposit Total) and the User Value. Here this amount will normally either be posted as an Overage or Shortage amount depending on the totals entered by the user.

 

Please take note of the following on the Cashier Totals screen:

 

  • For any foreign currency pay method, the ‘System Value’ will appear in the actual foreign amount and NOT the local equivalent.

  • In this case the ‘System Total’ will include the foreign currency value, and not the local equivalent.

  • You should not be able to save any Cashier Totals if no values have been entered. If you attempt to save Cashier Totals without entering any values you should receive the prompt: “There are no Cashier Totals to save”.

Saving Cashier Totals

 

  • The entries will not be saved unless a reason is entered.

  • The supervisor should click on the save button, any differences should then be posted as an overage or shortage. An information box should appear to say that an Overage or shortage will be posted (displaying the correct value).

  • Overages will be posted to an 'overs' account (special account to be set up per country) - transaction type OVS - they can still be investigated and transferred out if necessary.

  • Shortages will be posted to an individual cashiers account and should be investigated.

  • You should receive a printer prompt, which should then print the Cashier’s Daily Reconciliation.

  • The following prompt should then appear: “Outstanding income must be deposited before other functions can be used”. On clicking on the OK button, you should be directly taken into the Cashier Deposits screen where you should be forced to process the Cashiers Deposits before you are able to use any other functionality.

Cashier’s Daily Reconciliation printout

 

The Cashier’s Daily Reconciliation printout should display the values as displayed on screen for each pay method for ‘System’, ‘User’, Deposit’, ‘Difference’, including the ‘Reasons’ which you entered for each pay method.

 

Below are the definitions of other fields towards the bottom of the print out:

 

  • Total Payments - should reflect the Total Income taken for each pay method including any corrections & refunds processed.

  • Net Receipts - should be the total of all income taken EXCLUDING any corrections and refunds

  • Corrections & Refunds fields - should display the total of all refunds and corrections processed by the cashier.

History tab – Cashier Totals History

 

This screen allows the cashier to see the history of their Cashier Totals.

 

  • The Cashier drop down should default to the cashier that is logged into the system.

  • Clicking on the 'Load' button should populate the ‘Cashier Totals History’ grid and display a history of the Cashiers Totals.

  • By double clicking on a line in the grid the user is able to view additional information for that particular Cashier Totals, for e.g. they may wish to check the differences and the reasons for the differences in the values entered previously for that total.

Branch tab

 

This screen shows all individual payments/refunds transactions for the branch, listed by employee.  

 

  • The ‘from’ and ‘to’ dates are changeable by the user.

  • When clicking on the ‘Load’ button, the transactions are loaded for the selected branch in between the ‘from’ and ‘to’ date specified.

  • Columns can be sorted by clicking on the column headings.

  • Clicking on the excel button should export the results correctly to excel.

Summary Report tab

 

This screen shows  the summed up values of the ‘Cashier Daily Totals’ at Branch or Company Wide including the sum of securitised values.

 

Under the Summary Report tab you should be able to:

 

  • Select a specific branch from the ‘Choose Branch’ drop down or

  • Select ‘All Branches’ from the drop down to view the company wide totals.

  • A ‘from’ and ‘to’ date should be selected to return the Cashier Daily Totals between these dates.

  • Clicking on the ‘Load’ button should retrieve the results and display them by pay method.

  • Clicking on the ‘Print’ button should print the Cashier Daily Totals Summary Report.

 

Return to index

 

Definitions of fields on the Summary Report tab:

 

  • Pay Method – List of pay methods

  • System Value column – should display the summed total of all payments taken for each pay method, minus any refunds or corrections processed on that pay method.

  • User Value column – should display the summed totals for each of the pay methods which the cashiers entered when entering their Cashier Totals.

  • Deposit Value column – should display the summed total of any ‘Allocated float – disbursements’ performed using that pay method.

  • A.S column – should display the total payment amounts taken against securitised accounts only for each of the pay methods (minus any refunds or corrections).
