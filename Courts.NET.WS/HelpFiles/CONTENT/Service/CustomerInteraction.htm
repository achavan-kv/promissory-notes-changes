Customer Interaction

This screen will search on a customer ID field or may be called from another screen passing the customer ID. When a customer is retrieved the customer search fields will become read only until either the ‘Save’ or ‘Clear’ button is pressed. The customer title, first name and last name will be populated in the one field called ‘Name’. This field will always be read only. All interactions with this customer will be listed in date order.

 

The Interaction Log will include the columns:

  • Date
  • Description
  • Employee number
  • Account No ~ can be blank
  • Service Request No ~ can be blank
  • Comments

 

The user will not be able to edit existing entries.

A new interaction will be added by entering a date and the interaction code.

The interaction description will be automatically populated from code maintenance.

The date and code fields will be mandatory. If this customer relates to an account number or to an SR number then these can be selected from the drop down lists.

Only the account numbers and the SR numbers that relate to this customer will be available in the drop down lists.

The plus button is pressed to add the entry to the log. Note that the details will not be saved to the database until the ‘Save’ button is pressed.

All of the accounts for the customer will be listed at the bottom of the screen.

This list will be made up of the columns:

  • Service request number ~ can be blank. There can be more than one SR held against the same account. If this is the case then the SR with the latest date logged will be shown in this list.
  • Account number
  • Date account opened • Account type
  • Agreement total
  • Outstanding balance
  • Arrears amount
  • Main holder or joint
  • Account status

Where an account has an SR the user will be able to right click the account line and select an option called ‘Service Request’ to open the Service Request screen. The Service Request screen will be passed the SR number. That screen should load all related SR’s for the same customer accounts and scroll the product list to the SR number that was passed.
