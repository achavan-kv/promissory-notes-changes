Beyond Economic Repair (BER)

This popup is called from the Service Request Screen when a product is BER. The BER calculation is only applicable if the product is under warranty and the total price of previous claims has exceeded a percentage of the retail price. The percentage can be set up in country maintenance under "Service BER Limit"

 

The Fields on this screen will be read only and passed in by the calling screen as follows:

  • Allowable repair total - Original Retail Price
  • Previous Repair total - Sum of any previous repairs excluding food loss
  • Repair Estimate - From the Product Tab on the Service Request screen. it can be blank.
  • Balance available - This is calculated as 'Allowable repair total' - 'Previous Repair total'

If the Repair Estimate is blank the buttons on this screen will be:

  • Issue Replacement - This will:
  1. This button is only enabled for a Courts Account
  2. Close the SR
  3. Tick the "Replacement" box on the Resolution tab. This shows the user that this item should be replaced via the Courts Identical Replacement or Exchange process.
  4. The Service Request Status will be set to 'R'. The SR will only be closed when the resolution date is set and saved.
  • Require Estimate - This will:
  1. The Allocation tab will be enabled and the Service Request will need to be allocated.
  2. Once Allocated, the technician will enter the repair estimate
  3. The BER screen will popup again. The user will then be able to allocate the SR from the BER screen.

If the Repair estimate had been entered the 'Require Estimate' button will be re labelled to 'Allocate'. The SR status will be set to 'A'.
