Service Request Screen

The Service Request screen will be used to raise and track each Service Request (SR)

 

The screen caters for 3 types of Service Requests:

  • Courts Accounts - Existing Courts customers bringing in a Courts item for repair
  • Non Courts Accounts - Non Courts customers bringing in an item for repair
  • Stock Repairs - Internal Repair to stock

When an SR is linked to a Courts or Non courts account the first three digits of the SR number will equal the number entered as the service branch

For stock repair the first 3 digits will be the head office branch.

The menu option available from the main screen are:

  • Save
  • Reopen
  • View Audit Trail
  • Print SR
  • Account details
  • Stock search

Courts Accounts

Creating an SR for Courts Accounts

There are various ways to create and SR for a Courts Customer.
  • The user may enter the customers account number
  • Press the search button next to the Acct No field to Search by Customer ID
  • Enter the Cash & Go invoice number if the customer bought the goods on Cash & Go
What happens after the Customer Account is found?
  • The Customer and Product List tab will be populated with the details of the products on that account. This will only included delivered items. Where there are multiple quantities of one product, this will show as separate lines for each item.
  • Non Stock items will be excluded from the products list tab.
  • The product tab (the tab at the bottom of the screen next to transit notes, allocation, resolution, charge to analysis and comments tabs), will be populated with details of any product selected from the Product List. For more on product tab click here.
  • The customer details on the customer tab will be read only. There is a "notes" button which will toggle the address fields with the note box containing directions to the customers address. Amending customer details: Click on the handshake button in the Customer tab. This will open the customer record screen. Note that old SR's for existing Customers should show the customer details as at the date of resolution of that particular SR
What should the columns in the Product List display?
  • SR Number - If an SR has already been raised, otherwise it will be blank.
  • Date SR Logged - Will be blank if an SR has not been raised yet.
  • Status
    1. N = New. This is when a SR has been raised and not yet allocated to a technician
    2. D =New and Deposit Required. Once allocated the status will be set to 'A'
    3. E = Estimate Required. This is if the product invokes the Beyond Economic Repair Pop up and the user has selected the option to continue with the repair of this item.
    4. A = Allocated. This is when an SR is allocated to a technician and the SR has not yet reached the repair date. The repair date is set in the allocation screen.
    5. R = Resolved. This happens when the SR has reached it's repair date. If the charge to party is NOT Customer or Deliverer, then the status will automatically be set to 'C' (closed)
    6. C = Closed. If the charge to party was Customer or Deliverer, the status will only be set to closed once the resolution date is set AND the outstanding balance for the repair has been settled. If the charge to party was the Supplier then the status is set to closed after the repair date has passed and the resolution date is set.
  • Product Code
  • Sequence Number - Starts at 1 and increments for the multiple quantity of the same item.
  • Product Description 1 & 2
  • Delivery note number
  • Delivery date
  • Extended Warranty - This will be set to yes if the warranty has entered its extended period. This will only be after the Manufacturer Warranty period has expired. This field is for Supa Shield Warranties. Not Instant Replacement. There is an Instant replacement indicator on the Product tab below the customer Details and Product List tabs.
  • Contract Number - This will show the current contract number associated with the item. If the warranty has been renewed, the contract number will be the one for the Renewal Warranty.
Explain Create, Re Open and Audit Buttons?
  • Create New SR:
    1. This will not be enabled if there is already an SR number against the product and the status is NOT set to "closed". If it is closed the user will be able to create a new SR against the item in the event that the item is bought in more than once for repairs.
    2. A unique SR number will be generated for the new SR.
    3. The status for this product will be set to 'N'. More on status click here.
    4. Be sure to enter the correct service branch when creating the SR. This will default to the branch the user is logged into.
  • Re Open SR:
    1. From the Service Request Menu the user can select to "ReOpen Service Request". This will only be enabled if there is an SR Number against the product and the status is closed.
    2. Once the reopen option is ticked the user may only enter comments on the SR.
    3. If the item has been bought in for repairs and it previously had an SR number against it, the user will be able to create a new SR against that item. The previous repair on that item would have to be closed.
  • View Audit: 
    1. This will open the Service Request Audit Screen. 

Non Courts Account

Creating Non Courts Accounts Service Requests

See More

For non Courts customers both the customer and the product must be entered before a new SR can be raised. The customer may be on the system if they have bought in other items for previous repairs in the past.

The user will be able to search by customer ID or create a new customer from the Service Request screen.

If the customer had previously bought in items for repair these will be visible in the Product List.

Here are the steps:

  • Press "Create New SR"
  • The user will be forced to enter the following details in the product tab:
    1. Purchase Date
    2. Product code
    3. Stock location
    4. Description of the product

  • Date logged - this will default to Today but can be changed to an earlier date.

    The remaining fields are entered in the same way as if creating a Courts Account.

Stock Repair

Creating Stock Service Requests

Important things to know about Stock SR's
  • The customer ID for Stock repairs will always be Courts. This is created automatically.
  • The customer details on the customer tab will be blank except for the name "Courts"
  • The will be no account created for these repairs
  • Searching for stock repairs is done by entering the SR number
  • The service branch will default to head office and it will be invisible.
  • Entering product details in the same as for non courts accounts.

Transit Notes

This a free text comments field for users to add or amend comments. The field is intended to describe the location of the product between the customer and repairer

Comments

The top half of the comments screen is read only. The bottom area is used to add new comments and once saved will be viewable in the read only section along with the date, time and employee number of the person who entered the comment.
