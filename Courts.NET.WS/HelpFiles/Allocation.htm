Service Request Allocation

The allocation tab is only open when:

  • The SR is open AND
  • The fields required for allocation have been completed on the product tab AND
  • The Service Request status is either 'D', 'N', or 'A' AND
  • the current date is before the repair date or the repair date is blank AND
  • the resolution date has not yet been set.

Once in the allocation screen proceed to allocate the repair to a technician.

The user will have to set up zones, technicians and the technician diary before being able to allocate in this screen. This is done in the technician maintenance screen.

 

Technician Maintenance

This screen will allow the entry and update of technicians.

 

When this screen is open the user will see the list of existing technicians. There is a 'New' button if the user wishes to create a new technician.

To see technician details, double click a line in the list. You can now update the details for the selected technician.

When the new button is pressed the detail fields will be enabled to allow the entry of a new technician.

The following fields are mandatory:

  • Title
  • First Name
  • Last name
  • Telephone
  • Zones - These must be set up in codes maintenance (SRZONES)
  • Hours works - These are the daily hours the technician works and must be written in 24 hour format.
  • Calls per day - this will determine the number of slots in the technicians diary. If this value is ever reduced the system will check whether there are any booked slots in the future outside the new range.

The "Apply to all " button will apply the details of the selected technician to all technicians

There is also a vacation section to block out days when the technician will not be available for work.

The box on the right shows all the SR's allocated to the technician and the status and labour cost of each. Right clicking this box give a short cut to the SR Screen.

The "delete" button will only be enabled if there are no open SR's and no labour cost is owed to the technician.

 

If parts are required then a parts expected date can be entered.

 

The next step is to book a slot for the repair to be done. Press "Diary" to schedule a date and slot for this repair.

Technician Diary

This where a repair date and slot are booked.

 

This screen is accessed from the Allocation Tab through pressing the diary screen or via the main service menu.

Once a repair / estimate date has been entered by the diary any change to the technician selected must un-book the diary entry and clear the repair / estimate date. It will only be possible to change the parts expected date to a date later than the repair date by first using the diary to clear the repair date or to book a later repair date. When a repair / estimate date is entered the following fields will become mandatory:

  • Zone
  • Technician
More Details on the Technician Diary:

 

A drop down field will list the technicians so that one may be selected.

 

When this screen is invoked from the main menu the Service Request Number field near the top right of the screen will be invisible and the technician field will default to blank.

 

When invoked from the allocation tab the Service Request Number will be visible and read only.

 

The allocation tab will pass the Service Request Number and the technician (if entered) to this screen which will initialise these fields. The user may change the technician, in which case the new technician will be passed back to the allocation tab.

 

If a ‘Parts Expected’ date has been entered on the allocation tab the date fields on this screen will not allow a date before the ‘Parts Expected’ date.

 

The date fields on this screen will default to the later of the current date or the ‘Parts Expected’ date.

 

The user may specify a date when availability is required in the ‘Filter on Availability’ field. When the adjacent search button is pressed the technician drop down will be re-populated only with technicians with at least one slot available on the entered date.

 

The bottom section of the screen displays the calendar for one week from a start date entered in the ‘Availability from’ date field. This date field will default to the filter date above. The user may change the start date if they wish.

 

Two grid fields will be used to display the slots for the morning and afternoon. The number of slots per day will be determined by the setup in technician maintenance.

If there is an odd number of slots per day, the odd slot will be shown in the morning.

 

Booked slots will be displayed in a different colour. Clicking on a slot will display the details made up of the SR number, the slot number and a repair / estimate radio field.

 

A different fill colour will be used to distinguish repairs from estimates.. Where multiple slots are booked to the same SR, clicking on any one slot will allow all of the slots for the same SR to be unbooked. The ‘Free’ button will only be enabled for a booked slot(s).

 

It will be possible to view historic dates. The slot details, the ‘Book’ button and the ‘Free’ button must be disabled for a historic date.

 

Note that this screen will not be available for any technician marked as deleted by the technician maintenance screen. A technician can only be marked as deleted in that screen if there are no outstanding SRs allocated to the technician.
